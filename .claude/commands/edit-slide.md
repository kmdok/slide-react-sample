# Edit Slide Command

このコマンドは既存のスライドを修正するためのものです。

## 手順

1. **既存のスライド一覧を取得してください**:

   - `src/slides/index.tsx` を読み込んで、既存のスライド一覧を取得
   - 各スライドのファイル名とコメント（スライドの説明）を抽出
   - スライドの現在の順序も把握

2. **AskUserQuestion ツールを使って修正するスライドを選択してください**:

   **質問1: 修正するスライド**
   - header: "スライド選択"
   - question: "どのスライドを修正しますか？"
   - options: 既存のスライド一覧（例: "TitleSlide - タイトルスライド"）
   - multiSelect: false

3. **選択されたスライドファイルを読み込んでください**:

   - `src/slides/{選択されたスライド名}.tsx` を Read ツールで読み込む
   - 現在の内容を把握

4. **AskUserQuestion ツールを使って修正内容を選択してください**:

   **質問2: 修正項目**
   - header: "修正内容"
   - question: "何を修正しますか？"
   - options:
     - "タイトルを変更" (description: "スライドのタイトルテキストを変更")
     - "背景スタイルを変更" (description: "背景色やグラデーションを変更")
     - "コンテンツ全体を書き換え" (description: "スライドの中身を完全に書き直す")
     - "コンテンツを追加" (description: "既存の内容に追加でコンテンツを加える")
   - multiSelect: false

5. **選択された修正項目に応じて、追加の質問をしてください**:

   **タイトルを変更する場合**:
   - "新しいタイトルを入力してください"
   - ユーザーに自由入力させる

   **背景スタイルを変更する場合**:
   - header: "背景"
   - question: "新しい背景スタイルを選択してください"
   - options:
     - "青グラデーション" (description: "bg-gradient-to-br from-blue-500 to-purple-600 text-white")
     - "緑グラデーション" (description: "bg-gradient-to-br from-green-500 to-teal-600 text-white")
     - "白背景" (description: "bg-white text-gray-800")
     - "ダークモード" (description: "bg-gray-900 text-white")
     - "カスタム" (description: "TailwindCSSクラスを直接指定")

   **コンテンツ全体を書き換える場合**:
   - "どんな内容にしますか？"
   - ユーザーに自由入力させる
   - その内容に基づいて新しいJSXコンテンツを生成

   **コンテンツを追加する場合**:
   - "どんな内容を追加しますか？"
   - ユーザーに自由入力させる
   - 既存のコンテンツの後に追加

6. **Edit ツールを使ってスライドファイルを修正してください**:

   - 選択された修正内容に応じて、適切な部分を Edit ツールで更新
   - タイトル変更: `<h1>` タグの中身を変更
   - 背景変更: `className` 属性を変更
   - コンテンツ変更: `<Slide>` タグの children 部分を変更

7. **TypeScript チェックとビルドを実行してください**:
   ```bash
   pnpm exec tsc --noEmit && pnpm run build
   ```

8. **完了メッセージを表示してください**:
   - 修正したスライドのファイルパス
   - 変更内容の要約
   - スライドの表示位置（何番目）
   - アクセス用URL（例: http://localhost:5173/2）

## 注意事項

- **既存コンテンツの保持**:
  - コンテンツを追加する場合は、既存の内容を削除しないように注意してください
  - Edit ツールで old_string を正確に指定してください

- **タイトル変更の注意**:
  - h1, h2 など複数の見出しがある場合は、どれを変更するか明確にしてください
  - 必要に応じてユーザーに確認してください

- **背景変更の注意**:
  - text-white や text-gray-800 など、テキストカラーも背景に応じて適切に設定してください
  - グラデーションと単色背景でテキストの可読性を確保してください

- **コンテンツ生成の注意**:
  - ユーザーの要望に基づいて、適切な HTML 構造を生成してください
  - TailwindCSS クラスを使って美しいレイアウトを作成してください
  - リスト、画像、コードブロックなど、必要な要素を適切に配置してください

- **確認**:
  - 大きな変更を行う前に、変更内容をユーザーに確認してください
  - Edit が失敗した場合は、エラー内容をユーザーに伝えて対処方法を相談してください
